1) Computers

By size, purpose, and architecture. Types include mainframes, minicomputers, microcomputers, desktops, and workstations.

Mainframe

Room-sized systems for massive, secure, concurrent processing.

Minicomputer

Midrange, multi-user systems (legacy, still seen in labs).

Microcomputer

Personal/home computing with microprocessors.

Desktop

General-purpose, upgradable, non-portable PCs.

Workstation

High-performance pro systems (CAD, video, science).

Hierarchy Chart

1. Mainframe Computers

Banks, governments, airlines; mission-critical workloads.

Use: Large-scale processing (transactions, census, reservations)

Tech: High-performance CPUs, massive parallelism

OS: IBM z/OS, Linux on zSystems

Size: Room/large cabinets

Power: Exceptionally high (BIPS), thousands of users

Pros: Reliability, security, throughput

Cons: Very expensive, specialist staff

Price: $500k – $10M+

2. Minicomputers

Midrange, multi-user; legacy but relevant in certain orgs.

Use: SMB apps, databases, ERP; labs & manufacturing

Tech: Multi-user architectures

OS: UNIX, Linux, VMS

Size: Cabinets, smaller than mainframes

Power: Moderate–high (MIPS), multi-user

Pros: Cheaper than mainframes, multi-user

Cons: Limited scalability, declining

Price: $5k – $200k

3. Microcomputers

Personal/home computing (web, office, light gaming).

Use: Home/office basics

Tech: Intel Core, AMD Ryzen

OS: Windows, macOS, Linux

Size: Compact

Power: Low–moderate (MIPS)

Pros: Affordable, easy to use

Cons: Limited for intensive tasks

Price: $300 – $2,000

4. Desktop Computers

Stable, upgradable systems for work and play.

Use: Docs, coding, gaming

Tech: Core/Ryzen + discrete GPU

OS: Windows/macOS/Linux

Size: Fits on/under desk

Power: Moderate–high (BIPS)

Pros: Customizable, long life

Cons: Not portable, needs peripherals

Price: $500 – $3,000+

5. Workstations

Pro-grade (CAD, video, simulation), ECC memory.

Use: Video editing, CAD, scientific modeling

Tech: Xeon/Threadripper + Pro GPUs

OS: Windows Pro, macOS, Linux

Size: Bulkier (cooling/power)

Power: Very high (tens of BIPS)

Pros: Reliable (ECC), multi-monitor

Cons: Expensive, needs expertise

Price: $2,000 – $10,000+

2) Mobile Devices & Related

Categories, pros/cons, and real-world applications. Convergence: multiple functions in one device.

Hierarchy Chart

graph TB; A[Mobile and Related] --> S[Smartphones]; A --> R[Rugged or Specialist Phones]; A --> T[Tablets]; A --> C[Consumer Electronics]; C --> C1[Cameras and Camcorders]; C --> C2[Game Consoles]; C --> C3[Home Entertainment]; C --> C4[Media Players]; A --> N[Navigation Devices]; A --> M[Multifunction or Converged];

Smartphones

Versatile (camera, GPS, internet, apps). Indispensable for comms, entertainment, productivity.

  • Apps: social, banking, shopping, work
  • Cons: battery life, fragility, cost

Specialist / Rugged Phones

Durable, long battery, simple UI for focused tasks.

Use: construction, outdoors, seniors

Tablets

Touch-first, portable media and productivity devices.

Use: reading, e-learning, streaming, notes, casual gaming

Consumer Electronics

  • Cameras/Camcorders: pro capture; bulky/expensive vs phones
  • Game Consoles: high-fidelity gaming (PS/Xbox/Switch)
  • Home Entertainment: smart TVs, home theatre
  • Media Players: MP3/iPods (offline/long battery)

Navigation Devices

Dedicated GPS units (Garmin, in-car). Accurate, but often replaced by phone GPS apps.

Use: logistics, fleet, hiking, travel

Multifunctional / Converged Devices

Combine several features in one (smartphone/tablet). Saves devices/cost; may be less specialized.

3) Features of Digital Devices

Hierarchy Chart

graph TB; A[Features] --> P[Portability]; A --> Perf[Performance]; Perf --> CPU[CPU Speed]; Perf --> RAM[RAM]; Perf --> StorSpd[Storage Speed]; Perf --> SW[Software Efficiency]; A --> Stg[Storage]; Stg --> H[HDD]; Stg --> S[SSD]; Stg --> U[USB or SD]; Stg --> C[Cloud]; A --> UI[User Interfaces]; UI --> CLI[CLI]; UI --> Menu[Menu-driven]; UI --> GUI[GUI WIMP]; UI --> Voice[Voice]; UI --> Gesture[Gesture]; A --> Media[Media Support]; Media --> Formats[File Formats]; Media --> Codecs[Codecs and Drivers]; Media --> MedTypes[Storage Media]; A --> Energy[Energy Consumption]; Energy --> Efficient[Efficient Hardware]; Energy --> Sleep[Sleep or Hibernate]; Energy --> AutoOff[Auto Shutdown and Off]; A --> Exp[Expansion Capability]; A --> Sec[Security]; Sec --> SoftSec[Software Security]; Sec --> PhysSec[Physical Security]; A --> Conn[Connectivity];

Portability

Ease of carrying/using a device in different places.

Portable

  • Laptop: school notes/assignments
  • Tablet: client presentations
  • Smartphone: travel navigation
  • USB drive: move lesson files

Non‑Portable

  • Desktop: fixed desk
  • TV: living room
  • Printer: office spot

Performance

How quickly and efficiently tasks run.

  • CPU speed: more instructions/sec
  • RAM: more/faster RAM = better multitasking
  • Storage speed: SSD faster than HDD
  • Software efficiency: optimized code uses fewer resources

Q1: Processor speed

3 GHz = 3 billion instructions/s.

3e9 × 10 s = 30,000,000,000 instructions

Q2: RAM capacity

8 GB = 8192 MB, each program 500 MB.

8192 ÷ 500 = 16.384 → 16 programs

Q3: Virtual memory

If 4 GB more needed beyond RAM → 4 GB virtual memory used.

Q4: SSD transfer

4 GB = 4096 MB at 550 MB/s.

4096 ÷ 550 ≈ 7.44 s

Q5: Multitasking

20 apps × 1 GB = 20 GB needs; RAM = 16 GB → 4 GB virtual memory required.

Storage (as a feature)

Permanent data retention (files, apps, OS).

  • More capacity = more files/apps
  • Faster storage = faster load/boot; better virtual memory
  • Types: HDD, SSD, USB, SD/microSD, cloud

User Interfaces

CLI

Text commands; fast for experts, steep learning curve.

Windows CMD, macOS Terminal, Linux

Menu-driven

Simple navigation through menus; slower steps.

ATMs, printers, car settings

GUI (WIMP)

Windows, Icons, Menus, Pointers; easy but resource-heavy.

Windows, macOS, Android/iOS apps

Voice

Hands-free; misinterpretations possible.

Siri, Alexa, Google Assistant

Gesture

Touch/motion-based; intuitive but can be finicky.

Smartphones, tablets, Kinect

Media Support

  • File formats: Images (.jpg, .png), Audio (.mp3, .wav), Video (.mp4, .avi), Documents (.pdf, .docx)
  • Storage media: SD/microSD, DVD, USB, external HDD
  • Codecs & drivers: compression/decompression and hardware compatibility

Energy Consumption

  • Devices: PCs, monitors, printers, servers, charging mobiles
  • Impact: higher energy → higher emissions
  • Reduce: efficient hardware (LED, SSD), sleep/hibernate, auto-shutdown, cloud, switch off
  • Green computing: minimize power and e‑waste

Expansion Capability

  • Internal (PC): RAM, GPU, extra drives
  • External: USB devices, HDMI displays
  • Mobile: microSD (on supported devices)

Security Features

Software Security

  • Passwords/PIN, timeouts, retry limits
  • Biometrics (Touch ID, Face ID)
  • Protects data, prevents unauthorized access

Physical Security

  • Cable locks, secure rooms
  • Tamper-resistant screws
  • Protects the device itself

Connectivity

Networking, wired/wireless standards, internet access, device pairing (covered in Unit 2).

Keyboard

Use: writing, logins, coding. Pros: precise. Cons: fatigue.

Mouse (Pointing)

Use: GUI navigation, drag/select, drawing. Needs flat surface.

Microphone

Use: voice commands, recording, calls. Noise affects accuracy.

Touchscreen

Use: phones, tablets, kiosks/ATMs. Intuitive; smudges possible.

Biometric Scanner

Use: unlock, secure access, attendance. Privacy considerations.

Card Reader

Use: ATM, POS, access control, transport. Cards can wear.

Sensors

Types: light, motion, temperature, proximity. Automation benefits.

ii) Output Devices

Monitor

  • Types: LCD/LED, OLED, Touch
  • Resolution: HD, FHD, 4K
  • Use: docs, media, design, gaming
  • Cons: eye strain, cost for large 4K

Printer

  • Inkjet: great photos; slower
  • Laser: fast text; cost-effective for volume
  • 3D: layered models; slower, costly

3D Printers – Quick Debate

For: creativity, education, medical, SMEs

Against: safety/IP, cost, environment, skill

Plotter

High-precision vector output (engineering, maps, blueprints). Expensive, slower than large-format printers.

Data Projector

Classrooms/conferences; LCD/DLP. Large display; bulbs cost; lighting affects quality.

Speakers

Audio output (music, video, VR). Wireless options; quality varies; can disturb others.

Control Devices (Actuators)

Motors, valves, heaters, coolers, pistons; automate physical actions in systems.

Hierarchy Chart – Media vs Devices

Primary Storage

  • RAM: volatile, fast read/write, holds active apps/data
  • ROM: non‑volatile, stores firmware/BIOS

Secondary Storage

  • Non‑volatile; retains data when off
  • Holds files/apps; can serve as virtual memory
  • Internal (HDD/SSD) or external (USB/optical)

Media vs Devices

  • Media (stores data): magnetic (HDD/tape), optical (CD/DVD/BD), solid‑state (flash)
  • Devices (read/write): HDD/SSD, ODD, USB drives, card readers
  • Memory jogger: Media = where data lives; Device = what accesses it.
